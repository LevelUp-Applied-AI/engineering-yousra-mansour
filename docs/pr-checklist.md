# PR Self-Review Checklist

Before opening a pull request, I confirm the following:

- [ ] The code does what the PR title and description claim
- [ ] All tests pass locally
- [ ] New functionality includes appropriate test coverage
- [ ] The README and documentation are accurate and updated
- [ ] The changes are focused on one clear objective
- [ ] There are no debug statements (print, breakpoint, logs)
- [ ] Code follows project style and naming conventions
- [ ] No commented-out or unused code remains
